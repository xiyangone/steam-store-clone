# Steam 官网风格复刻与自动部署提示词

请基于当前项目，稳定复刻 Steam 官网首页的核心视觉与交互，目标是让页面在布局、色彩、层级、视频背景、排行榜区、底部页脚、按钮、文字样式、间距和响应式表现上都尽量接近官网。

要求如下：

1. 技术栈保持 React + TypeScript + Vite，不随意更换框架。
2. 优先复用现有项目结构，在原有组件基础上重构，不做无关大改。
3. 首页顶部 Hero 区必须支持全屏背景视频展示，表现尽量接近 Steam 官网，仅保留视频主视觉，不叠加多余文案。
4. 排行榜区需要实现与 Steam 官网接近的双栏结构：左侧榜单、右侧详情预览、顶部标签切换、底部“查看更多”，并补齐接近官网的内容数据。
5. 页脚区域需要对齐 Steam 官网的层级、灰色文字、图标、注册链接、登录按钮、上下间距和整体背景氛围。
6. 所有样式要兼顾桌面端和移动端，避免高度错位、文字换行异常、左右栏不齐平等问题。
7. 数据结构要可维护，排行榜、截图、价格、标签、平台、评分等内容统一收敛到 mock 数据或配置层。
8. 修复过程中要优先定位根因，不能用临时补丁掩盖问题。
9. 每次修改后都要执行必要验证，至少包含构建验证；若涉及页面行为，还要补充浏览器检查。
10. 最终需要提供 GitHub Pages 自动部署方案，保证仓库推送后可自动发布线上预览。

自动化部署要求：

1. 配置 GitHub Actions，在推送到 `main` 分支后自动执行构建与 GitHub Pages 部署。
2. Vite 的 `base` 需要兼容 GitHub Pages 项目路径。
3. 仓库的 Pages 发布源使用 `GitHub Actions`，不要依赖工作流自动创建 Pages 站点。
4. 部署完成后输出最终线上访问地址，并确保页面静态资源路径正常。

执行要求：

1. 先分析现有代码结构，再给出最小但完整的修改方案。
2. 直接产出代码，不停留在泛泛建议。
3. 明确列出修改文件、关键改动、验证命令、部署结果。
4. 若发现与 Steam 官网仍有明显差异，继续迭代到接近为止。
